Golang字符串

ASCII表

ASCII（American Standard Code for Information Interchange，美国信息交换标准代码）是基于拉丁字符的一套字节编码系统

注意要熟记下面表格中的特殊字符

序号 转义字符 十进制数 备注
1 \x00 0 null字符，是表中的第一项，在C语言中是字符串的结束符
2 \x09或\t 9 制表符tab
3 \x0a或\n 10 换行
4 \x0d或\r 13 回车
5 \x30 ~ \x39 48~57 字符0~9
6 \x31 49 字符1
7 \x41 65 字符A
8 \x61 97 字符a
9 \x20 32 空格

UTF-8、GKB都兼容了ASCII，Unicode又包括了ASCII

字符

本质上来说，计算机中的一切都是字节，字符串也是多个字节组合而成的，就是多个字节形成的有序序列。但是对于多字节编码的中文来说，用一个字节描述不了，需要多个字节表示一个字符，Go提供了rune类型。

  • byte：兼容ASCII码的字符，是byte类型，即uint8的别名，占用1个字节
  • rune：汉字等字符，Unicode编码，是rune类型的，即int32的别名，占用4个字节
  • 一个字符字面常量用单引号引起来

字符串与byte序列、rune序列转换

注意：

  • string(一个整数)，强制类型转换一个整数，相当于把整数当Unicode码点，在去查一个字符，最后返回的是一个字符串。
  • string(整数序列)，强制类型转换一个整数序列，也是将整数当Unicode码点去查字符，最后也是转成字符串。
  • 这里是将整数当成unicode码点，再将得到的码点去查unicode码表，用码点找到表中对应的字符，在强制类型转换成字符串。

字符串

  • 是字面常量，只读，是不可变的
  • 线性数据结构，可以使用索引来检索
  • 值类型
  • 是utf-8编码

字符串长度

使用内建函数len，是返回的字符串所占用的字节数，时间复杂度为O(1)，字符串是字面常量，在定义时就知道了长度，并记录了下来到元数据中。

索引

  • 在go语言中是不支持负索引的，索引范围是[0-len(s)-1]
  • 即使字符串中有中文字符，索引指的是按照字节的偏移量来编的
  • 时间复杂度O(1)，使用索引计算该字符相对开头的偏移量即可
  • 对于顺序表来说，使用索引查找效率是最高的
  • s[i]是获取索引i处的UTF-8编码的一个字节

遍历

C风格使用索引遍历，这里相当于字节遍历

for-range遍历，安全的遍历字符

strings库

strings提供了大多数字符串的操作函数，使用方便。

注意：字符串是字面常量，不可修改，基本上操作都是返回的是新的字符串。

拼接

  • strings.Join：使用特定间隔拼接字符串切片
  • strings.Builder：多次拼接，推荐使用
  • fmt.Sprintf：方便快捷，推荐使用
  • 简单拼接字符串常用+、fmt.Sprintf
  • 假设手里正好有字符串的序列的话可以考虑使用join
  • 如果要频繁多次拼接的话，strings.Builder是推荐的方式
  • strings.Builder和bytes.Buffer这两个用法是相同的

查询

  • strings.Index：从左至右搜索，返回子串第一次出现的字节索引位置。没有找到的话就返回-1，子串为空时返回的是0。
  • strings.LastIndex：从右至左搜索，返回子串第一次出现的字节索引位置。没有找到的话就返回-1。
  • strings.IndexByte、strings.IndexRune与strings.Index类似；strings.LastIndexByte与strings.LastIndex类似。
  • strings.IndexAny：从左至右搜索，找到给定的字符集字符串中任意一个字符就返回索引位置。未找到返回-1。
  • strings.Contains方法本质上就是Index方法，只不过返回bool值，方便使用bool值时使用。
  • strings.LastIndexAny与strings.IndexAny搜索方向相反。
  • strings.Count：从左至右搜索字串，返回字串出现的次数。

时间复杂度是O(n)，效率不高，不过该用的时候还是要用，但是要少用。

大小写

  • strings.ToLower：转换为小写
  • strings.ToUpper：转换为大写

时间复杂度O(n)

前后缀

  • strings.HasPrefix：是否以子串开头
  • strings.HasSuffix：是否以子串结尾

时间复杂度O(1)，效率高

移除

  • strings.TrimSpace：去除字符串两端的空白字符。
  • strings.TrimPrefix、strings.TrimSuffix：如果开头或结尾匹配，则去除。否则，返回原字符串的副本。
  • strings.TrimLeft：字符串开头的字符如果在字符集中，则全部移除，直到碰到第一个不在字符集中的字符为止。
  • strings.TrimRight：字符串结尾的字符如果在字符集中，则全部移除，直到碰到第一个不在字符集中的字符为止。
  • strings.Trim：字符串两头的字符如果在字符集中，则全部移除，直到左或右都碰到第一个不在字符集中的字符为止。

分割

  • strings.Split：按照给定的分割子串去分割，返回切割后的字符串切片。
    • 切割字符串是被切掉了，是不会出现在结果中
    • 没有切到，也就会返回一个元素的切片，元素就是被切的字符串
    • 分割字符串为空串时，那么返回将被切割字符串按照每个rune字符分解后转成string存入切片返回
  • strings.SplitN(s, sep string, n int) 返回的类型是[]string ，n表示最终切片中有几个元素
    • n == 0，返回空字符串，切成0个子串
    • n > 0，返回切片元素的个数
      • n == 1，返回一个元素的切片，元素为z，相当于啥也没切到返回一个元素的切片
      • n > 1，按照切割符切割。返回多个元素的切片。按照切割符切成的段数最多有x段，当n < x 时，会有部分剩余字符串没有切；当n == x 时，字符串s正好从头切到尾切完，返回所有段的切片；当n > x时，和n == x是一样的。n表示切割出来的子串的上限，也就是切片里面至多有n个元素。
    • n < 1，相当于Split，能切多少就切出多少。
  • strings.SplitAfter和strings.Split相似，strings.SplitAfter就是不把切割符切掉。
  • strings.SplitAfterN和strings.SplitN相似，strings.SplitAfterN也是不把切割符切掉。
  • strings.Cut(s string, sep string) (before string, after string, found bool)
    • 内部使用Index找切割符，所以是从左至右搜索切割点，可以认为就是切一刀，一刀两段
    • 没有切到，返回 原字符串，false
    • 切到了，匹配切割符部分要切掉，返回 切割符前部分，切割符后部分，true

替换

  • strings.Replace(s string, old string, new string, n int) string
    • n < 0，等价于ReplaceAll，指的是全部都替换
    • n == 0，或old ==new，就是返回原字符串
    • n > 0，至多替换n次，如果n超过找到old子串的次数x，也就只能替换x次了
    • 没有找到替换处，就返回原字符串

其他

  • strings.Repeat(s string, count int) string
    • 使用给定的字符串重复n次拼接成一个新的字符串
  • strings.Map(mapping func(rune) rune, s string) string
    • 按照给定处理每个rune字符的函数依次处理每个字符后，在拼接成字符串返回。要注意Map是一对一的映射，不能减少元素个数。

类型转换

数值类型转换

  • 低精度向高精度转换可以，高精度向低精度转化会损失精度
  • 无符号向有符号转换，最高位是符号位
  • byte和int可以相互转换
  • float和int可以相互转换，float到int会丢失精度
  • bool和int不能相互转换
  • 不同长度的int和float之间可以相互转换

原码、反码、和补码是计算机中三种常见的编码方式。

  • 原码（Sign-Magnitude）：原码是最为直观的表示方式，其中最高位是表示符号位（0表示正数，1表示负数），其余位表示数值的绝对值。例如：+7 的原码是00000111，-7的原码是10000111.
  • 反码（Ones’ Complement）：反码是通过对原码取反得到的，符号位不变，正数的反码与原码相同，负数的反码是将原码中的后7位取反，例如：+7 的反码00000111，-7 的反码11111000.
  • 补码(Two’s Complement)：补码是对反码进行加1操作得到的，正数的补码与原码是相同的，而补码是在反码的基础上再加1；最高位任然还是表示的符号位，例如：+7 的补码是00000111，-7 补码是11111001.

如果上面不使用无类型常量，有类型的计算的话类型不一致是会报错的，因为Go是对类型要求非常苛刻的语言，需要强制类型转换。

类型别名和类型定义

原因是在源码中定义了type byte = uint8，这里的byte就是uint8的别名。

在Go v1.9版本开始就使用了类型别名，将byte等的定义修改了类型别名的方式。

uint8的别名说明就是uint8的另外一个名字罢了，和uint8是一回事。
