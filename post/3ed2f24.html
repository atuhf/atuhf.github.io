MongoDB

MongoDB属于非关系数据库，它是由C++编写的分布式文档数据库，内部使用类似于JSON的bson二进制格式。

中文文档：https://www.w3cschool.cn/mongodb/

安装

下载地址：https://www.mongodb.com/try/download/community

在下载地址中下载对应操作系统的MongoDB，并运行它。

使用Windows的话可以下载官方的zip包，解压完就可以使用了。

组件 文件名
Server mongod.exe
Router mongos.exe, Query Router, Sharding Cluster
Client mongo.exe
MonitoringTools mongostat.exe, mongotop.exe
ImportExportTools mongodump.exe, mongorestore.exe, mongoexport.exe, mongoimport.exe
MiscellaneousTools bsondump.exe, mongofiles.exe, mongooplog.exe, mongoperf.exe

运行

选项说明：

  • –bind_ip ip 逗号分隔IP地址。默认localhost
  • –bind_ip_all 绑定所有本地IP地址
  • –port port 端口，默认27017
  • –dbpath path 数据路径，缺省为 \data\db\ 。windows下缺省就是当前盘符的根目录
  • –logpath path 指定日志文件，替代stdout，说明默认是控制台打印日志
  • -f file 指定配置文件，yaml格式
  • 注册windows服务
    • –install 注册windows服务
    • –serviceName name 服务名称
    • –serviceDisplayName name 服务显示名

配置文件

MongoDB配置是使用的yaml文件格式

  • 嵌套使用缩进完成，不支持Tab等制表符，支持空格
    • 缩进空格数不限制，但是只要同一级元素对其即可
  • 冒号后面要有空格间隔
  • 大小写敏感
  • #表示注释
  • 字符串不需要引号，有特殊符号的时候可以使用引号
  • 布尔
    • true、True、TRUE、yes、YES都是真
    • false、False、FALSE、no、NO都是假
  • null、Null、~波浪线都表示为空，不指定的值的话默认也是为空

yaml参考文档：https://www.w3cschool.cn/iqmrhf/dotvpozt.html

MongoDB配置参考：https://mongoing.com/docs/reference/configuration-options.html

配置选项说明

  • storage
    • dbPath，必须指定MongoDB的数据目录，这个目录必须存在
  • systemLog
    • destination，缺省是输出日志到std，file表示输出到文件
    • logAppend，true表示在已经存在的日志文件追加。默认为false，每次启动服务，重新创建新的日志。
    • path，日志文件存放路径，文件目录必须存在
  • net
    • port，端口，缺省值为27017，客户端连接使用
    • bindIp，缺省绑定是到127.0.0.1地址上面，这个根据情况修改

身份验证机制

官方文档：https://www.mongodb.com/docs/drivers/go/current/fundamentals/auth/

支持的机制

Go 驱动程序支持以下身份验证机制：

  • SCRAM-SHA-256
  • SCRAM-SHA-1
  • MONGODB-CR
  • MongoDB-AWS
  • X.509

Go Driver 通过身份验证机制建立连接 客户 类型。该Client类型指定用作连接选项的机制和凭据。凭据类型。要配置这些选项，请将Credential类型传递给设置验证()的方法客户选项类型。

约定示例

每个身份验证机制都包含以下占位符：

  • username MongoDB用户名
  • password MongoDB用户密码
  • hostname MongoDB服务器网络地址，可由您的客户端访问
  • port MongoDB服务器端口号
  • authenticationDb 包含用户身份验证数据的MongoDB数据库。如果省略此选项，驱动程序将使用默认值admin。

默认机制

默认机制使用以下身份验证机制之一，具体取决于您的服务器支持的 MongoDB 版本：

基本概念

Go Driver的使用，官方博客参考：https://www.mongodb.com/blog/post/mongodb-go-driver-tutorial

MongoDB中是可以创建使用多个库的，但是有一些数据库名是保留的，可以直接访问这些有特殊作用的数据库。

  • admin：从权限角度来看，这就是”root”数据库，要是将一个用户添加到这个数据库中，添加的用户会自动继承所有数据库的权限。一些特定的服务器端命令也只能从这个数据库中运行，比如列出所有的数据库或者关闭服务器。
    • admin数据库是MongoDB的管理数据库，只有管理员权限的用户才可以访问；
    • 在admin数据库中，可以对用户进行管理、权限设置、数据库备份和还原等操作；
    • 一般情况管理员会在admin数据库下创建其它数据库的用户，并为这些用户分配相应的权限。
  • local: 这个数据永远不会被复制，可以用来存储限于本地单台服务器的任意集合 。
    • local数据库是与分片相关的数据库，但它只会在每个分片上存在；
    • local数据库用于存储分片节点的复制集信息和本地操作日志；
    • 当一个节点作为分片副本集的成员时，它会在local数据库中记录与副本集相关的信息。
  • config: 当Mongo用于分片设置时，config数据库在内部使用，用于保存分片的相关信息。
    • config数据库时用于支持MongoDB的分片（Sharding）功能的配置数据库；
    • 当MongoDB部署为分片集群时，config数据库中会存储与分片有关的配置信息，如分片键、分片节点等；
    • 分片配置服务器会维护config数据库中的信息，用于管理数据的分片和迁移。

BSON

https://www.mongodb.com/docs/drivers/go/current/fundamentals/bson/

将Go类型转换为BSON称为编组，而相反的过程称为解编组。

Go驱动程序提供了四种主要类型来处理BSON数据：

  • D: An ordered representation of a BSON document (slice)，表示有序的，切片且元素是二元的
  • M: An unordered representation of a BSON document (map)，表示无序的，map且元素是kv对
  • A: An ordered representation of a BSON array，BSON数组的有序表示
  • E: A single element inside a D type，D 类型中的单个元素

示例：如何使用 bson.D 类型来构造查询筛选器，以匹配quantity字段值超过100的文档

查询数据

单条查询

bson.D{{"name", "tom"}}

  • bson.D是切片，D后的{}表示切片的字面量定义
  • {“name”, “tom”}表示一个结构体实例字面量定义
    • “name”是结构体的Key属性，类型是string
    • “tom”是结构体的Value属性，类型是any

bson.M{"name": "tom"}

  • bson.M是map，M后面{}表示该map的字面量定义
  • map的类型是map[string]any

查询条件

修改上面findMany2的函数，将filter改为用传参的方式来给查询条件，可以使用下面表中的不同的filter来传参实现查询。

比较符号 含义 filter示例(这里用age字段来演示)
$lt 小于 bson.M{"age": bson.M{"$lt": 20}}
$gt 大于 bson.M{"age": bson.M{"$gt": 20\}}
$lte 小于等于 bson.M{"age": bson.M{"$lte": 20}}
$gte 大于等于 bson.M{"age": bson.M{"$gte": 20}}
$eq 等于,一般都不会用这个符号 bson.M{"age": bson.M{"$eq": 20}} bson.M{"age": 20}
$ne 不等于 bson.M{"age": bson.M{"$ne": 20}}
$in 在范围内 bson.M{"age": bson.M{"$in": []int{13,18,33}}}
$nin 不在范围内 bson.M{"age": bson.M{"$nin": []int{13,18,33}}}

https://www.mongodb.com/docs/manual/reference/operator/query/and/#examples

逻辑符号 含义 filter示例
$and 与 bson.M{"$and": []bson.M{{"name": "tom"}, {"age": 33}}} bson.M{"$and": []bson.M{{"name": "tom"}, {"age": bson.M{"$gt": 20}}}}
$or 或 bson.M{"$or": []bson.M{{"name": "tom"}, {"age": 30}}} bson.M{"$or": []bson.M{{"name": "tom"}, {"age": bson.M{"$gt": 20}}}}
$not 非 bson.M{"age": bson.M{"$not": bson.M{"$lte": 18}}}

bson.M{"age": bson.M{"$lte": 18}} 取反的话可以写成 bson.M{"age": bson.M{"$gt": 18}} 或者写成 bson.M{"age": bson.M{"$not": bson.M{"$lte": 18}}}

元素 含义 filter示例
$exists 判断文档中是否存在这个字段 bson.M{"Age": bson.M{"$exists": true}}
$type 判断字段是否是指定的类型 bson.M{"age": bson.M{"$type": 16}}
